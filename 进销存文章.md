# 二、三张表跑通全流程 

对于进销存的管理，核心就三张表，货品库存台账、入库表和出库表。这三张表覆盖了从物品管理到出入库的全流程。

## 1.货品库存台账 

记录物品的“基本信息” 维护 SKU 基础信息：如物品编号（唯一ID）、物品名称、规格型号、剩余库存、货品照片。 ![pic_918044b2.png](https://ideaflow-article-to-markdown.hf.space/images/pic_918044b2.png)

# 三、三张表之间的关系 

这一点要注意，因为这三张表并不是互相独立的， 而是有关联关系的。比如做入库/出库，是要选择货品库存台账里的物品的，另外，填写了入库和出库数量时，对应库存信息表里的物品库存也会相应的增加和减少。 所以我们需要用多维表格的关联字段、查找引用和公式来实现这些效果。 比如： ①入库时，我们需要选择物品编号，物品编号字段选择“单向关联”类型，关联的数据表就是库存台账表 这个时候当我们点击字段，会有一个弹框，这个弹框就是库存信息表里的内容。我们选择需要入库的物品 ![pic_5de695bb.png](https://ideaflow-article-to-markdown.hf.space/images/pic_5de695bb.png) ②库存计算 还有一个重点，计算库存。 在库存信息表中，剩余库存的字段，用来展示目前物品剩余的库存数 ![pic_22bd8213.png](https://ideaflow-article-to-markdown.hf.space/images/pic_22bd8213.png) 剩余库存的计算逻辑很简单，实际上就是这个表里的“当前累积入库数量 - 当前累积出库数量” 我们用公式字段，写一个公式就行 ![pic_6b77a4f3.png](https://ideaflow-article-to-markdown.hf.space/images/pic_6b77a4f3.png) 以上几步做完之后，这个库存管理应用就算搭建起来了，可以开始正常使用了。   


# 四、体验进阶 

这里就会经常有朋友会问， 感觉还是在用一个excel表，虽然比excel表多了自动化和权限的功能，但还是很难上手。其实你面对小白用户也很难解释多维表格和excel的区分，只能花时间去教他们上手使用。 这里我们可以用多维表格的应用模式，给表格外面“包一层皮”，让它看起来更像系统。更符合用户的使用习惯，上手也快。 ![pic_92e6f056.png](https://ideaflow-article-to-markdown.hf.space/images/pic_92e6f056.png) 看完之后其实我们也会感受到应用模式的优势： 1.界面更友好 ①自定义页面布局，不再是密密麻麻的表格 ②大按钮、大字体，手机操作也不费劲 ③图标化展示，一眼就能看懂 2.操作更简单 ①常用功能放在显眼位置 ②复杂操作简化成一键完成   


# 四、在上线前我们还需要想清的三个问题 

1.  谁日常操作？ 一般是库管或运营。
2.  新货第一次怎么进系统？ 台账建档 → 入库单 → 库存增加。
3.  出库怎么防漏？ 用自动化：出库完成给领用人一条飞书确认，重要物料可再加审批。

再往下，用仪表盘给老板看滞销、缺货、周转；用工作流做低库存提醒——和 CRM 一样，都是多维表格的强项。 用飞书搭系统，最难的不是技术，是“懂自己的业务”。先想清楚“我需要系统解决什么问题”，再用表格和应用模式把需求“翻译”成功能。 不妨从建第一张“数据表”开始，不用一步到位，先搭个能用的，再慢慢优化。最好的系统不是最复杂的，而是最适合你业务的那一个。
