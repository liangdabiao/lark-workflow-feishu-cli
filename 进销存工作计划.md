# 进销存 ERP Skill 工作计划

## 一、项目概述

基于飞书多维表格（Bitable）构建一套轻量化进销存 ERP 系统，以 lark-cli workflow skill 的形式封装，用户通过自然语言即可完成货品管理、入库、出库、库存预警、数据分析等全流程操作。

**核心原则**：三张表跑通全流程（货品库存台账 + 入库表 + 出库表），通过关联字段、公式字段实现库存自动计算，零代码、多协同。

---

## 二、整体架构

```
lark-workflow-erp-inventory (Skill)
│
├── 数据层：飞书多维表格 (Bitable)
│   ├── 货品库存台账 (products)     — SKU 主数据
│   ├── 入库记录表 (stock_in)       — 入库流水
│   ├── 出库记录表 (stock_out)      — 出库流水
│   └── 供应商表 (suppliers)        — 供应商信息（扩展）
│
├── 计算层：公式字段 / Lookup 字段
│   ├── 剩余库存 = 累计入库 - 累计出库
│   ├── 库存预警 = IF(剩余库存 ≤ 最低库存, "低库存", "正常")
│   └── 货品关联（入库/出库 → 台账）
│
├── 交互层：用户自然语言指令
│   ├── "新建一个进销存系统"  → 初始化 Base + 创建表结构
│   ├── "入库：米色T恤 10件" → 写入库记录
│   ├── "出库：米色T恤 3件"  → 写出库记录（校验库存）
│   └── "查库存 / 库存报表"  → 查询 + 聚合分析
│
└── 通知层：飞书消息 / 自动化
    ├── 低库存预警通知
    ├── 出库确认通知
    └── 定期库存报表推送
```

## 三、数据表设计

## 六、关键技术点

### 1. 库存计算（公式字段）

```
剩余库存 = 入库汇总数 - 出库汇总数

实现方式：
- 入库表：关联字段(→台账) + 数量字段
- 出库表：关联字段(→台账) + 数量字段
- 台账表：公式字段 = SUMIF(入库表.关联物品=当前记录, 入库数量) - SUMIF(出库表.关联物品=当前记录, 出库数量)
```

### 2. 关联字段配置

```
入库表.关联物品 → type="link", foreign_table="products"
出库表.关联物品 → type="link", foreign_table="products"
```

### 3. Lookup 字段配置

```
入库表.物品名称 → lookup from stock_in.关联物品, select products.物品名称
出库表.物品名称 → lookup from stock_out.关联物品, select products.物品名称
```

### 4. 出库校验逻辑

```
出库前校验：
1. 获取物品当前剩余库存（从台账表读取）
2. 比较：剩余库存 >= 出库数量 ?
3. 不足 → 提示用户并中止
4. 充足 → 执行出库记录写入
```

### 5. 限流与分页

```
- +table-create 后等待 1 秒再创建下一张表
- +field-create 每次间隔 1-2 秒
- +record-list 每页最多 200 条，用 offset 分页
- +record-upsert 串行写入，间隔 0.5-1 秒
- +xxx-list 类命令禁止并发
```

## 十、风险与应对

| 风险 | 影响 | 应对策略 |
|------|------|---------|
| 公式字段创建依赖字段 ID | 初始化流程复杂 | 先建存储字段，获取字段 ID 后再创建公式字段 |
| 字段创建限流 | 初始化可能部分失败 | 串行创建 + 间隔 1-2 秒 + 失败重试补字段 |
| 关联字段类型约束 | Lookup 依赖关联先建好 | 严格按顺序：存储字段 → 关联字段 → Lookup → 公式 |
| 大批量数据操作 | 请求超时/限流 | 分批处理，每批 ≤ 50 条，间隔 1 秒 |
| 并发操作冲突 | 数据不一致 | 所有写操作串行执行 |
