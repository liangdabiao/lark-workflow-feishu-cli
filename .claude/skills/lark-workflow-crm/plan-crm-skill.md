# 微型团队 CRM Skill 全面规划

> 基于 `微型crm.md` 参考文档，面向小团队销售/客服场景，基于飞书多维表格 + lark-cli 的轻量 CRM 系统。

---

## 一、Skill 基本信息

| 项目 | 内容 |
|------|------|
| **名称** | `lark-workflow-crm` |
| **版本** | 1.0.0 |
| **定位** | 面向小团队的微型销售 CRM（客户管理 + 商机漏斗 + 跟进记录 + 自动化提醒） |
| **触发词** | "CRM"、"客户管理"、"商机跟进"、"销售漏斗"、"客户建档"、"赢单丢单"、"跟进提醒"、"销售日报" |
| **身份要求** | `--as user`（个人操作）或 `--as bot`（自动提醒），初始化用 user |
| **依赖** | `lark-cli`，domain: `base,im,contact` |

### 与现有 personal-crm 的区别

| 维度 | personal-crm（已有） | crm（本 skill，新建） |
|------|---------------------|----------------------|
| **定位** | 个人关系管理 | 团队销售管理 |
| **核心对象** | 联系人 + 互动记录 | 客户 + 商机 + 跟进记录 |
| **核心流程** | 导入联系人→算热度→提醒 | 建档→商机→跟进→阶段流转→赢单/丢单 |
| **数据来源** | 邮箱/日历自动扫描 | 手动录入 + AI 辅助 |
| **看板** | 无 | 商机阶段看板（Kanban） |
| **仪表盘** | 无 | 销售漏斗 + 转化率 + 阶段分布 |
| **自动化** | 冷联系人提醒 | 阶段变更通知 + 超时跟进 + 赢单报喜 |

## 二、数据模型（3 张表）

### 表 1：客户信息主表 (customers)

客户的基础档案，所有商机的入口。

| 字段名 | 类型 | 说明 |
|--------|------|------|
| 客户名称 | 文本 | 个人客户填姓名，企业客户填公司名 |
| 客户类型 | 单选 | 个人客户 / 企业客户 |
| 联系人 | 文本 | 对接人姓名 |
| 手机号 | 文本 | 联系电话 |
| 邮箱 | 文本 | 邮箱地址 |
| 行业 | 单选 | 互联网/金融/教育/制造业/零售/其他 |
| 公司规模 | 单选 | 1-10人/11-50人/51-200人/200人以上 |
| 来源渠道 | 单选 | 主动咨询/转介绍/展会/线上广告/社群/其他 |
| 地址 | 文本 | 所在城市/地区 |
| 负责人 | 人员 | 销售/对接人（Feishu user） |
| 重要程度 | 单选 | 高/中/低 |
| 客户状态 | 单选 | 活跃/沉睡/流失 |
| 备注 | 文本 | 自由备注 |
| 创建时间 | 日期 | 首次建档时间（系统字段或手动） |
| 最后跟进时间 | 日期 | 最近一次跟进的时间（公式或手动更新） |

### 表 2：商机管理表 (opportunities)

核心漏斗表，每个客户可以有多个商机。

| 字段名 | 类型 | 说明 |
|--------|------|------|
| 商机名称 | 文本 | 如"XX公司 ERP 系统采购" |
| 关联客户 | 关联 | 双向关联到「客户信息主表」 |
| 商机金额 | 数字 | 预计成交金额（元） |
| 商机阶段 | 单选 | 待接触 / 需求确认 / 方案报价 / 商务谈判 / 赢单 / 丢单 |
| 负责人 | 人员 | 商机跟进负责人 |
| 预计成交日期 | 日期 | 预计关单时间 |
| 竞争对手 | 文本 | 主要竞品或对手 |
| 赢单概率 | 数字 | 0-100%，按阶段自动建议：待接触10%/需求确认30%/方案报价50%/商务谈判75%/赢单100%/丢单0% |
| 丢单原因 | 单选 | 价格/功能/竞品/无预算/项目取消/其他 |
| 商机来源 | 单选 | 新开发/老客户增购/转介绍 |
| 创建时间 | 日期 | 商机创建时间 |
| 关单时间 | 日期 | 赢单或丢单的时间（公式：当阶段=赢单或丢单时自动记录） |

### 表 3：跟进记录表 (follow_ups)

每次客户沟通的详细记录，替代微信群/Excel日报。

| 字段名 | 类型 | 说明 |
|--------|------|------|
| 关联客户 | 关联 | 双向关联到「客户信息主表」 |
| 关联商机 | 关联 | 双向关联到「商机管理表」（可选） |
| 跟进方式 | 单选 | 电话/微信/面访/邮件/视频会议/其他 |
| 跟进摘要 | 文本 | 沟通内容摘要 |
| 客户反馈 | 文本 | 客户的态度和反馈 |
| 下一步计划 | 文本 | 下次跟进的动作 |
| 下次跟进时间 | 日期 | 约定的下次联系时间 |
| 跟进人 | 人员 | 执行跟进的人 |
| 商机阶段变更 | 单选 | 无变更/推进到下一阶段/退回上一阶段/赢单/丢单 |
| 跟进时间 | 日期 | 跟进发生时间 |

### 表 4：团队目标表 (team_targets)（可选增强）

用于销售目标管理和绩效统计。

| 字段名 | 类型 | 说明 |
|--------|------|------|
| 负责人 | 人员 | 销售/团队成员 |
| 月份 | 单选 | 2026-01 ~ 2026-12 |
| 目标金额 | 数字 | 当月销售目标（元） |
| 实际金额 | 公式 | 通过 Lookup 从商机表汇总（阶段=赢单 AND 关单时间在本月） |

## 三、表间关联设计

```
customers ◄──────► opportunities（双向关联）
     │                       │
     │                       │
     └──────── follow_ups ───┘（两个独立关联字段）
```

- **customers ↔ opportunities**：双向关联，一个客户可对应多个商机
- **customers ↔ follow_ups**：双向关联，跟进记录关联到客户
- **opportunities ↔ follow_ups**：双向关联，跟进记录可选关联到商机

          │            │            │            │         │

## 六、自动化提醒设计

通过 lark-cli 的定时任务或 Cron 实现以下自动化：

### 自动化 1：每日跟进提醒（每日 9:00）

```
触发：每天早上 9 点
动作：
  1. 查询「下次跟进时间 = 今天」的跟进记录
  2. 查询「商机阶段 ≠ 赢单/丢单」且「预计成交日期 = 今天」的商机
  3. 通过 im +messages-send 发送飞书消息给对应负责人
消息格式：
  📋 今日跟进提醒
  - 【客户A】下次跟进时间到，计划：XXX
  - 【商机B】预计今天成交，金额：XXX
```

### 自动化 2：超时未跟进预警（每日 10:00）

```
触发：每天早上 10 点
动作：
  1. 查询商机表中「最后跟进时间距今 > 7天」且「阶段 ≠ 赢单/丢单」的商机
  2. 发送提醒给负责人
消息格式：
  ⚠️ 商机超时提醒
  - 【商机A】已 8 天未跟进，阶段：方案报价
  - 【商机B】已 12 天未跟进，阶段：需求确认
```

### 自动化 3：赢单报喜（实时/定时检查）

```
触发：阶段变更为「赢单」时
动作：
  1. 发送飞书群消息通知全团队
消息格式：
  🎉 赢单喜报！
  - 商机：XX公司 ERP 采购
  - 金额：¥100,000
  - 负责人：张三
  - 商机周期：XX天
```

### 自动化 4：每周销售简报（每周一 9:00）

```
触发：每周一早上 9 点
动作：
  1. 汇总上周新增客户数、新增商机数、赢单数、赢单金额
  2. 汇总当前漏斗各阶段商机数量和金额
  3. 发送给指定群或负责人
消息格式：
  📊 周销售简报（上周）
  - 新增客户：5 家
  - 新增商机：8 个，总金额 ¥500,000
  - 赢单：3 个，金额 ¥200,000
  - 当前漏斗：待接触 12 → 需求确认 8 → 方案报价 5 → 商务谈判 3
```

## 七、核心操作模式

### 模式一：系统初始化

首次使用时，一键创建完整 CRM 系统：

```
步骤 1: 创建 Base（base +base-create）
步骤 2: 创建 3 张空表（customers / opportunities / follow_ups）
  - 注意：先建空表，不加字段（避免限流）
  - 每张表创建间隔 2 秒
步骤 3: 逐个添加字段（+field-create）
  - 每个字段间隔 1.5 秒
  - 先加存储字段，再加关联字段
  - 最后加公式/Lookup 字段
步骤 4: 创建双向关联字段
步骤 5: 创建公式字段（赢单概率建议、客户最后跟进时间等）
步骤 6: 创建 Lookup 字段（累计商机数、累计赢单金额等）
步骤 7: 创建视图（看板、我的商机、跟进时间线等）
步骤 8: 创建仪表盘及各组件
步骤 9: 保存 base_token 和各 table_id 到配置
```

### 模式二：客户建档

通过自然语言录入客户信息：

```
用户输入："帮我建一个客户：北京星辰科技有限公司，对接人王总，手机138xxxx，互联网行业，100人左右，来自展会，我负责，重要程度高"
AI 处理：
  1. 解析提取各字段值
  2. 调用 base +record-upsert 写入 customers 表
  3. 返回确认信息
```

### 模式三：商机创建与跟进

```
用户输入："给北京星辰科技创建一个商机：企业IM系统采购，预计金额50万，阶段待接触"
AI 处理：
  1. 查找或创建客户记录
  2. 调用 base +record-upsert 写入 opportunities 表
  3. 关联客户
```

```
用户输入："跟进记录：今天电话联系了王总，对方对方案比较满意，约下周三面访确认细节"
AI 处理：
  1. 创建跟进记录到 follow_ups 表
  2. 关联客户和商机
  3. 更新商机的最后跟进时间（如有需要）
  4. 如果用户提到阶段推进，同步更新商机阶段
```

### 模式四：商机漏斗查询

```
用户输入："看看当前的销售漏斗"
AI 处理：
  1. 调用 base +data-query 按商机阶段聚合
  2. 生成漏斗图文本展示

用户输入："本月赢单了多少？"
AI 处理：
  1. 筛选本月赢单商机
  2. 汇总金额和数量
```

### 模式五：销售日报

```
用户输入："生成今天的销售日报"
AI 处理：
  1. 查询今天所有跟进记录
  2. 查询今天新增的客户和商机
  3. 查询今天阶段变更的商机
  4. 生成结构化日报，可选发送到飞书群
```

### 模式六：数据统计与分析

```
用户输入："这个月各负责人的业绩排行"
AI 处理：
  1. 调用 base +data-query 按负责人分组聚合赢单金额
  2. 返回排行榜

用户输入："商机转化率怎么样？"
AI 处理：
  1. 按阶段统计商机数量
  2. 计算各阶段转化率
```

## 八、公式字段设计

### 公式 1：赢单概率建议（商机表）

```
IF(商机阶段="待接触", 10,
  IF(商机阶段="需求确认", 30,
    IF(商机阶段="方案报价", 50,
      IF(商机阶段="商务谈判", 75,
        IF(商机阶段="赢单", 100, 0)))))
```

### 公式 2：商机天数（商机表）

```
DATETIME_DIFF(NOW(), 创建时间, "DAY")
```

### 公式 3：客户最后跟进时间（客户表）

```
通过 Lookup 从 follow_ups 表取 MAX(跟进时间)
```

## 九、Lookup 字段设计

### Lookup 1：客户商机数量（客户表）

```
来源表：opportunities
关联字段：关联客户
聚合方式：COUNT
```

### Lookup 2：客户累计赢单金额（客户表）

```
来源表：opportunities
关联字段：关联客户
筛选条件：商机阶段 = "赢单"
聚合方式：SUM(商机金额)
```

### Lookup 3：商机跟进次数（商机表）

```
来源表：follow_ups
关联字段：关联商机
聚合方式：COUNT
```

## 十、限流与错误处理

| 场景 | 策略 |
|------|------|
| +table-create 限流 | 不带 --fields 参数，先建空表 |
| +field-create 限流 | 每个字段间隔 1.5-2 秒 |
| +view-create 限流 | 每个视图间隔 2 秒 |
| +dashboard-block-create 限流 | 每个组件间隔 1.5 秒 |
| +record-upsert 批量写入 | 每批最多 500 条 |
| +data-query 不支持计算字段 | 只用存储字段做筛选和聚合 |
| 字段创建部分失败 | 捕获错误，用 +field-list 检查已有字段，补创建缺失字段 |
