# 微型团队 CRM

基于飞书多维表格 + lark-cli 构建的轻量团队销售 CRM 系统。适合小团队管理客户档案、商机漏斗、跟进记录，配自动提醒和销售仪表盘。

## 快速开始

### 1. 确保已授权

```bash
lark-cli auth login --domain base,im,contact
```

### 2. 一句话初始化

直接用自然语言告诉 AI：

```
帮我创建一个CRM系统
```

AI 会自动完成以下步骤：
- 创建多维表格 Base
- 创建 3 张数据表（客户信息、商机管理、跟进记录）
- 添加所有字段（客户名称、商机阶段、跟进方式等）
- 配置 3 组双向关联（客户↔商机、客户↔跟进、商机↔跟进）
- 创建 6 个公式字段（赢单概率、商机天数、商机数量等）
- 创建视图（商机阶段看板、跟进时间线、本月赢单）
- 创建仪表盘（销售漏斗、赢单金额、阶段分布等 6 个组件）
- 创建客户咨询登记表单

初始化大约需要 3-4 分钟（因限流保护，每个字段间隔 1.5 秒）。

### 3. 开始使用

```
帮我建一个客户：北京星辰科技，对接人王总，手机138xxxx
```

## 定时自动化

可以在 Claude Code 中配置定时任务自动执行 CRM 相关操作：

### 每日跟进提醒（工作日 9:00）

```
CronCreate: cron="0 9 * * 1-5", prompt="检查今日需要跟进的客户和商机，发送提醒给对应负责人", recurring=true
```

### 超时未跟进预警（工作日 10:00）

```
CronCreate: cron="0 10 * * 1-5", prompt="检查超过7天未跟进的商机，发送预警提醒", recurring=true
```

### 每周销售简报（每周一 9:00）

```
CronCreate: cron="0 9 * * 1", prompt="生成上周销售简报，汇总新增客户、赢单情况、漏斗数据，发送到销售群", recurring=true
```

### 每日销售日报（工作日 18:00）

```
CronCreate: cron="0 18 * * 1-5", prompt="生成今天的销售日报，发送到销售群", recurring=true
```

## 商机阶段说明

| 阶段 | 含义 | 建议概率 |
|------|------|---------|
| 待接触 | 新线索，尚未与客户建立联系 | 10% |
| 需求确认 | 已接触，正在了解和确认需求 | 30% |
| 方案报价 | 需求明确，已提供方案和报价 | 50% |
| 商务谈判 | 方案认可，进入价格/合同谈判 | 75% |
| 赢单 | 成交 | 100% |
| 丢单 | 流失 | 0% |

## 与个人 CRM 的区别

| 维度 | 个人 CRM（personal-crm） | 团队 CRM（本 Skill） |
|------|------------------------|---------------------|
| 定位 | 个人关系管理 | 团队销售管理 |
| 核心对象 | 联系人 + 互动记录 | 客户 + 商机 + 跟进记录 |
| 核心流程 | 导入联系人→算热度→提醒 | 建档→商机→跟进→阶段流转→赢单/丢单 |
| 数据来源 | 邮箱/日历自动扫描 | 手动录入 + AI 辅助 |
| 看板 | 无 | 商机阶段看板（Kanban） |
| 仪表盘 | 无 | 销售漏斗 + 转化率 + 阶段分布 |
| 自动化 | 冷联系人提醒 | 阶段变更通知 + 超时跟进 + 赢单报喜 |

两个 Skill 可以同时使用，互不影响。

## 与其他 Skill 的关系

- **数据层**：依赖 `lark-base`（多维表格操作）
- **通知层**：依赖 `lark-im`（飞书消息发送）
- **人员层**：依赖 `lark-contact`（通讯录人员查询，用于查找负责人）
- **独立使用**：不依赖其他工作流 Skill，可单独使用
