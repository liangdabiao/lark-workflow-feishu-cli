## 项目简介

本项目的核心思想：**用飞书作为统一的数据底座和交互界面，用 Claude Code Agent /OpenClaw 作为智能调度中枢，把原本分散在 Gmail、Telegram、Todoist、Slack、Asana 等十几个 SaaS 工具中的个人效率工作流，全部收敛到飞书一个平台内运行。**

每个工作流被封装为一个 **Claude Agent/OpenClaw Skill**（SKILL.md），通过自然语言对话或指令触发，调用 `lark-cli` 操作飞书 API 完成数据读写、消息通知、文档生成等操作。

## 环境要求

| 组件 | 要求 |
|------|------|
| Claude Code/OpenClaw | CLI 工具或者 OpenClaw，用于执行 Agent 和 Skill |
| lark-cli | 飞书命令行工具，已全局安装 |
| 飞书应用 | App ID `cli_a94d22643c38dbcd`，需在开发者后台配置 |
| 用户授权 | 已通过 `lark-cli auth login` 完成登录授权 |
| Node.js | lark-cli 依赖 |

## 快速开始

### 1. 确认环境就绪

```bash
# 检查 lark-cli 版本
lark-cli --version

# 检查登录状态
lark-cli auth login --domain im,calendar,task,mail,base,docs,wiki,drive,vc,minutes

# 检查 Skill 是否就位
ls .agents/skills/lark-workflow-*/
```

### 2. 选择一个 Skill 开始

每个 Skill 首次使用时会自动引导你完成数据初始化（创建多维表格等）。直接用自然语言触发即可：

```
# 想看今天的安排
你：今天有什么安排？

# 想记录饮食
你：午餐吃了红烧牛肉面和一杯冰美式

# 想保存一篇文章
你：帮我保存这篇文章到知识库 https://example.com/article
```

### 3. 按需初始化

首次使用某个 Skill 时，AI 会提示你创建所需的多维表格。你也可以主动初始化：

```
你：帮我初始化个人CRM系统
你：初始化健康日记
你：设置晨间简报
```

### 1. 个人 CRM

> 自动扫描飞书邮箱和日历，提取联系人并维护关系热度分，支持自然语言查询和超期提醒。

**文件位置：** `.agents/skills/lark-workflow-personal-crm/SKILL.md`

**触发方式：** 对话中说以下任意关键词

```
"联系人管理"  "CRM"  "谁联系过我"  "多久没联系"  "联系人热度"  "关系管理"
```

**使用示例：**

```
你：帮我建立个人CRM系统
AI：首次使用需要创建数据表，现在开始初始化...
    [创建多维表格：联系人表 + 互动记录表]

你：从邮箱导入过去一个月的联系人
AI：正在扫描邮箱...发现 15 个新联系人，23 条互动记录。
    已自动过滤广告邮件和系统通知。

你：上个月谁联系过我？
AI：[查询互动记录表]
    上个月你与以下 8 人有互动：
    1. 张三（XX科技）— 邮件 3 次，最后 3/25
    2. 李四（YY投资）— 会议 1 次 + 邮件 1 次，最后 3/20
    ...

你：XX公司的人我最后一次聊过什么？
AI：[按公司过滤查询]
    你与 XX 公司的人最后互动是在 3/20：
    人物：王五（YY投资，投资经理）
    互动类型：视频会议
    内容摘要：讨论了 Q2 预算和人员招聘计划

你：有没有超过3个月没联系的重要关系？
AI：[查询热度分 < 30 且重要程度 = 高]
    发现 3 个需要跟进的联系人：
    | 姓名 | 公司 | 最后联系 | 热度分 |
    |------|------|---------|--------|
    | 赵六 | ZZ集团 | 95天前 | 8  |
    | ... | ... | ... | ... |
```

**热度分计算规则：**

| 维度 | 权重 | 计算方式 |
|------|------|---------|
| 频率分 | 40分 | 过去90天互动次数，0次=0分，20次+=40分 |
| 近期分 | 40分 | 最后互动距今天数，7天内=40分，90天+=0分 |
| 重要度分 | 20分 | 高=20分，中=12分，低=4分 |

**数据表结构：**

**联系人主表 (contacts)**

| 字段 | 类型 | 说明 |
|------|------|------|
| 姓名 | 文本 | 联系人姓名 |
| 公司 | 文本 | 所在公司 |
| 职位 | 文本 | 职位/角色 |
| 邮箱 | 文本 | 邮箱地址 |
| 手机 | 文本 | 联系方式 |
| 来源 | 单选 | 邮件/日历/通讯录/手动 |
| 关系类型 | 单选 | 客户/合作伙伴/同事/朋友/行业人脉/其他 |
| 重要程度 | 单选 | 高/中/低 |
| 热度分 | 数字 | 0-100 |
| 最后联系时间 | 日期 | 最近互动时间 |
| 最后联系内容 | 文本 | 互动摘要 |
| 备注 | 文本 | 自由备注 |

**互动记录表 (interactions)**

| 字段 | 类型 | 说明 |
|------|------|------|
| 关联联系人 | 关联 | → 联系人主表 |
| 互动时间 | 日期 | 互动发生时间 |
| 互动类型 | 单选 | 邮件(发)/邮件(收)/会议/消息/电话/其他 |
| 互动摘要 | 文本 | 内容简述 |
| 涉及主题 | 文本 | 讨论的话题 |

**所需权限：** `mail:mail:read` `calendar:calendar.event:read` `contact:employee.id:readonly` `bitable:app` `bitable:app:read` `bitable:app:write` `im:message:send_as_bot`

### 2. 会议待办追踪

> 从飞书妙记中提取待办事项，通过飞书消息确认后自动创建飞书任务，定期检查完成情况。

**文件位置：** `.agents/skills/lark-workflow-meeting-todo/SKILL.md`

**触发方式：**

```
"会议待办"  "会后的待办"  "追踪待办"  "提取会议行动项"  "整理会议纪要待办"
```

**使用示例：**

```
你：帮我整理最近一周会议的待办事项
AI：[搜索过去7天的会议]
    找到 5 场会议，其中 3 场有纪要。

    正在提取待办...

    ## 会议：Q2产品规划会（3/28）
    ### 我方待办
    | # | 待办内容 | 建议截止日 |
    |---|---------|-----------|
    | 1 | 把产品路线图发给张三 | 本周五 |
    | 2 | 调研竞品定价策略 | 下周三 |
    ### 对方待办
    | # | 待办内容 | 承诺人 |
    |---|---------|--------|
    | 1 | 提供用户调研数据 | 李四 |

    请确认需要创建飞书任务的待办：
    1. 把产品路线图发给张三 ✓
    2. 调研竞品定价策略 ✓
    3. 跳过"李四提供用户调研数据"（对方待办）

你：确认 1 和 2
AI：已创建 2 个飞书任务：
    - [ ] 把产品路线图发给张三（截止：周五）
    - [ ] 调研竞品定价策略（截止：下周三）

你：检查一下会议待办的完成情况
AI：[查询飞书任务状态]
    | 待办 | 来源会议 | 状态 |
    |------|---------|------|
    | 发送产品路线图 | Q2规划会 | 已完成 ✓ |
    | 调研竞品定价 | Q2规划会 | 进行中 |
    | 回复设计稿反馈 | UI评审会 | 已过期 ⚠️ |
```

**四种工作模式：**

| 模式 | 说明 | 示例 |
|------|------|------|
| 批量提取 | 指定时间范围批量提取所有会议待办 | "整理本周的会议待办" |
| 单会议提取 | 只处理指定的一场会议 | "提取今天产品评审会的行动项" |
| 完成检查 | 检查待办完成情况，验证承诺兑现 | "会议待办都完成了吗" |
| 自动归档 | 超14天未完成的待办自动归档 | 自动执行 |

**学习模式：** 如果用户多次拒绝某类待办（如"会议纪要类记录"），AI 下次会自动跳过类似内容。

**数据表结构：会议待办追踪**

| 字段 | 类型 | 说明 |
|------|------|------|
| 会议主题 | 文本 | 来源会议标题 |
| 会议日期 | 日期 | 会议时间 |
| 待办内容 | 文本 | 提取的待办描述 |
| 待办类型 | 单选 | 我方待办 / 对方待办 |
| 负责人/承诺人 | 文本 | 谁来做 |
| 建议截止日 | 日期 | 根据上下文推断 |
| 确认状态 | 单选 | 待确认/已确认/已跳过/已完成/已归档 |
| 关联任务ID | 文本 | 飞书任务的 task_id |
| 会议ID | 文本 | vc meeting_id |

**所需权限：** `vc:meeting:read` `minutes:minutes:readonly` `docx:document:read` `task:task:write` `task:task:read` `im:message:send_as_bot` `bitable:app:read` `bitable:app:write` `mail:mail:read`

### 5. AI 商业顾问团

> 8 个 AI 专家角色每晚自动分析业务数据，汇总去重后发送编号清单到飞书，支持回复编号展开详细分析。

**文件位置：** `.agents/skills/lark-workflow-business-advisor/SKILL.md`

**触发方式：**

```
"商业分析"  "业务报告"  "顾问分析"  "夜间分析"  "业务建议"
```

**使用示例：**

```
你：帮我分析一下最近的业务状况
AI：[拉取 CRM、社交追踪、创意库、投资等所有可用数据]
    [8个AI顾问各自独立分析]
    [汇总去重排序]

    ## 今日商业顾问团报告（3/31）

    ### 总览
    今日共 6 条建议，来自 5 位顾问。
    数据来源：CRM(有)、社交追踪(有)、创意库(有)、投资(无)

    ### 建议清单
    1. 【关系】张三（XX科技）已 45 天未联系，上次讨论了合作意向，建议主动跟进。
    2. 【财务】投资组合本周下跌 5%，科技股拖累，建议检查 AAPL 持仓比例。
    3. 【内容】本周内容发布频率下降 40%，建议安排时间完成待执行选题。
    4. 【运营】7 个任务已过期未完成，其中 3 个超过 14 天。
    5. 【风险】竞品 YY 产品本周发布新功能，与产品路线图重叠。
    6. 【增长】Instagram 增长势头良好，建议加大投入。

你：第 3 条详细说说

```

```
CronCreate: cron="0 22 * * 1-5", prompt="运行商业顾问团分析，生成今日报告并发送到飞书", recurring=true
```


> CronCreate 创建的定时任务会在 Claude Code 运行时自动触发。如需持久化，请设置 `durable: true`。

## 常见问题

### Q: 首次使用某个 Skill 时报错怎么办？

A: 大概率是权限未授权。运行以下命令：

```bash
lark-cli auth login --domain <需要的模块>
```

或者按错误提示中的 `hint` 字段执行。

### Q: 数据存在哪里？

A: 所有 Skill 的持久化数据都存储在**飞书多维表格 (Base)** 中。首次使用时 AI 会引导你创建表格，你也可以指定已有的 Base。数据归属你的飞书账号，可在飞书中直接查看和编辑。

### Q: 如何更换接收通知的飞书群聊？

A: 对 AI 说"把通知发送到 XXX 群"，AI 会通过 `im +chat-search` 搜索群聊 ID。

### Q: Skill 之间数据不互通怎么办？

A: 确保：
1. 相关 Skill 都已完成数据初始化（创建了多维表格）
2. 使用同一个 Base 存储数据（AI 会提示你选择或创建）

### Q: 定时任务不执行怎么办？

A: 定时任务依赖 Claude Code 进程运行。确保 Claude Code 正在运行中。如需持久化定时任务，在创建时设置 `durable: true`。

### Q: 社交媒体和股票数据不准确？

A: 当前阶段这些数据通过 AI WebSearch 获取公开信息，可能存在延迟或不准确。你可以：
1. 手动录入数据（直接告诉 AI 数值）
2. 等待后续版本接入各平台官方 API

### Q: 如何删除/重置某个 Skill 的数据？

A: 直接在飞书中打开对应的多维表格，删除记录或整个表即可。重新使用时会自动引导重新初始化。
